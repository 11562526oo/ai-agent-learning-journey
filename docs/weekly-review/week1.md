**遇到的问题：**
- role的类型 
  -  system 系统角色 用来规定模型的身份、风格、规则。
  -  user   用户角色 表示你说的话
  -  assistant  助手角色 表示模型之前说的话。
- `user_input.lower()` 错写成了 `user_input_lower()`

**我的理解：**
- .env 用来保存 API Key、Base URL、Model 等配置，避免把敏感信息硬编码到代码里。
- 一个最基本的大模型调用流程是：读取配置 -> 创建客户端 -> 构造 messages -> 发起请求 -> 解析响应。
- 很多 AI 聊天的基本思路 ： 把当前问题 + 一部分历史对话，一起发给模型。
  - 真实产品一般会做上下文管理 
    - 直接带最近几轮消息
    - 截断旧对话
    - 对旧对话做总结
    - 把长期信息存到外部存储

- 多轮会话之后 列表 - 字段结构
messages = [
    {"role": "system", "content": "你是一个智能助手"},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好呀，有什么可以帮你？"},
    {"role": "user", "content": "你是谁？"},
    {"role": "assistant", "content": "我是一个智能助手，可以陪你聊天，也可以帮你解答问题。"},
    {"role": "user", "content": "今天天气怎么样？"},
    {"role": "assistant", "content": "如果你告诉我你所在的城市，我可以帮你分析天气情况。"}
]

**我的理解：**
- messages[0] 取的是第一条消息，通常是 system
- messages[-MAX_HISTORY:] 取的是最后几条消息，也就是最近的历史
- [messages[0]] + messages[1:][-MAX_HISTORY:] 的作用是：保留唯一的 system + 保留最近几条非 
- 格式要求不够稳定时，不是模型完全不懂，而是提示词的约束条件还不够明确。
